树的定义和相关术语

小技巧

  1. 树的定义是递归的
  2. 森林的定义，注意树和森林的递归定义，果然还是放到一起学习。
  3. 结点的深度是从根结点开始自顶向下逐层累加的，结点的高度是从叶结点开始自底向上逐层累加的
  4. 树的路径长度是指树根到每个结点的路径长度的总和，根到每个结点的路径长度最大值应是树的高度减一
  5. 注意与哈夫曼树的带权路径长度相区别

一般树

一般树的存储结构

双亲表示法、孩子表示法、孩子兄弟法

双亲表示法

这种表示方法中，以一组连续的存储单元存储树的结点，结点包括数据域data和父结点parent域。
这种存储结构利用了每个结点（除根以外）只有唯一的双亲的性质。在这种存储结构下，求结点的双亲十分方便，也很容易求树的根，但求结点的孩子时需要遍历整个结构。

孩子表示法

第一种：多重链表中的结点是同构的，若d为树的度。由于树中很多结点的度小于d，所以链表中有很多空链域，空间较浪费，不难推出，在一棵有n个结点度为k的树中必有n(k-1)+1个空链域。

第二种：多重链表中的结点异构，用degree记录当前结点的度，这种方式虽然比第一种节约了空间，但是异构影响了对整个数据结构的操作，整体上操作变得更加复杂

第三种：另一种办法是，把每个结点的孩子结点排列起来，看成是一个线性表，且以单链表做存储结构，则n个结点有n个孩子链表(叶子的孩子链表为空表)。而n个头指针又组成一个线性表，为了便于查找,可采用顺序存储结构。

第四种：在第三种的基础上，结合双亲表示法，将第三种的顺序存储结构部分换为双亲表示法的结构。

满二叉树、完全二叉树

若用链表存储一棵二叉树时，每个结点除数据域外，还有指向左孩子和右孩子的两个指针。在这种存储结构中，n个结点的二叉树共有__2n______个指针域，其中有____n-1____个指针域是存放了地址，有______n+1__________个指针是空指针。

存储结构

顺序存储结构

二叉树的顺序存储从1开始会具有标号规律

//顺序存储结构 
#define MAXSIZE 100
typedef ElemType BiTree[MAXSIZE];

顺序存储结构使用一组地址连续的存储单元来存储数据元素，为了能够在存储结构中反映出结点之间的逻辑关系，必须将二叉树中的结点依照一定的规律安排在这组单元中。对于完全二叉树，只要从根起按层序存储即可，依次自上而下、自左至右存储结点元素，即将完全二叉树上编号为﹔的结点元素存储在如上定义的一维数组中下标为i1的分量中。而一般二叉树则按照完全二叉树的形式存储，不存在的结点用0或者其他不引起歧义的数据标识，所以整体来说，二叉树的顺序存储结构空间利用率不高。

//复制一棵和T完全相同的二叉树
void copy(BinaryTree t,BinaryTree newTree) { 
    if(t==NULL) { //空树递归结束
        newTree=NULL;
        return;
    } else {
        newTree=new BinaryTreeNode;
        newTree->data=t->data;//复制根结点
        copy(t->left,newTree->left);//递归复制左子树
        copy(t->right,newTree->right);//递归复制右子树
    }
}
//计算二叉树T的深度
int depth (BinaryTree t) { 
    if(t==NULL) return 0;//空树，深度为0，递归结束
    else {
        int m=depth (t->left);//递归计算左子树的深度记为m
        int n=depth (t->right);//递归计算右子树的深度记为n
        if(m>n)return(m+1);//二叉树的深度为m与n的较大者加1
        else return(n+1);
    }
}
//统计二叉树t中结点的个数
int nodeCount(BinaryTree t) { 
    if(t==NULL) return 0;//如果是空树，则结点个数为0，递归结束
    else return nodeCount(t->left)+nodeCount (t->right)+1;//否则结点个数为左子树的结点个数+右子树的结点个数+1
}
先序、中序、后序遍历、DFS、BFS
快速求解二叉树的先序、中序、后序序列

根据二叉树遍历的递归算法，对节点的访问顺序是二叉树的逆时针包边。这个顺序是基础。暂且将其成为基础访问顺序。

按照基础访问顺序，结合递归算法特点即可得出序列。如，先序，按照基础访问顺序，判断该节点的左右子节点是否被访问，未被访问则输出该节点的信息。中序同上，判断该节点左子节点被访问，右子节点未被访问。后续，判断该节点左右节点均被访问。

以上，会发现一些规律，按照基础访问顺序，上述判断未被访问的条件可以去掉。

则，先序，等同于基础访问顺序，但是需要注意曾经访问过的节点应该从图中去掉去掉，因为一个节点只能输出一次信息。

中序，只需判断其节点左子节点是否被访问过就行，根据规律，只要当判断一个节点的左节点被访问过了，那么这个节点的左节点一定被输出了。这个左节点也应当被去掉。

后序，上述说到，未被访问的条件可以去掉，但是后续的要求便是左右子节点均被访问，所以无法简化。但是左右节点均被访问这个条件就意味着，当前节点之下所有后代节点均被访问切输出信息了。接就是说，如果我们把访问过的节点去掉，那么当前节点便会是终端节点。易知，第一个输出信息的节点必然是终端节点。所以如果我们当一个节点被访问，然后我们就去掉他的话，那么我们一直访问的是终端节点。结合基础访问顺序。可以得出，后序，就是在删除节点过程基础上，最左边的终端节点。

二叉树的遍历包括：先序遍历、中序遍历、后序遍历、层次遍历

其中先序遍历、中序遍历、后续遍历三者区别在于对每棵子树的根节点val访问顺序不同，而实质上都是按照先左后右，深度优先的方式遍历节点。

层次遍历是按照先左后右，广度优先的方式遍历节点。

另外，有时相关场景要求先右后左进行遍历，与先左后右思路上没有本质区别。

总而言之，先中后序遍历采用深度优先思想，深度优先算法采用栈实现，层次遍历使用广度优先思想，广度优先采用队列实现，而二叉树天然是递归的（定义本身就是递归定义），故深度优先算法又可以基于递归思想实现。

def post_order_traversal(root: TreeNode):
    if not root:
        return []

    stack1 = [root]
    stack2 = []

    while stack1:
        node = stack1.pop()
        stack2.append(node)

        if node.left:
            stack1.append(node.left)
        if node.right:
            stack1.append(node.right)

    while stack2:
        node = stack2.pop()

def bfs(root: TreeNode):
    if root is None:
        return
    queue = Queue()
    queue.put(root)
    while not queue.empty():
        root = queue.get()
        print(root.val, end=" ")
        if root.left is not None:
            queue.put(root.left)
        if root.right is not None:
            queue.put(root.right)

def dfs_recursive(root):
    result = []
    def dfs(node):
        if node is None:
            return
        result.append(node.value)  # 访问节点
        dfs(node.left)  # 递归访问左子树
        dfs(node.right)  # 递归访问右子树

    dfs(root)
    return result

def dfs_iterative(root):
    stack = [root]
    result = []

    while stack:
        node = stack.pop()
        if node:
            result.append(node.value)
            # 先压入右子树，再压入左子树，确保先访问左子树
            if node.right:
                stack.append(node.right)
            if node.left:
                stack.append(node.left)

    return result

线索二叉树
一般二叉树线索化后即为线索二叉树

有n个结点的二叉链表中必定存在n+l个空链域，因此可以充分利用这些空链域来存放结点的前驱和后继信息。试做如下规定:若结点有左子树，则其lchild域指示其左孩子，否则令lchild域指示其前驱;若结点有右子树，则其rchild域指示其右孩子，否则令 rchild域指示其后继。为了避免混淆，尚需改变结点结构，增加两个标志域。
由于线索二叉树构造的实质是将二叉链表中的空指针改为指向前驱或后继的线索，而前驱或后继的信息只有在遍历时才能得到，因此线索化的过程即为在遍历的过程中修改空指针的过程，可用递归算法。对二叉树按照不同的遍历次序进行线索化，可以得到不同的线索二叉树，包括先序线索二叉树、中序线索二叉树和后序线索二叉树。而对二叉树线索化时，有时需要一个辅助结点指向可能的二叉树的根结点，这个结点被称为头结点。所以线索化可以分为无头结点的和有头结点的。

void inThreading (threadedBinaryTree t) {
    //pre是全局变量，初始化时其右孩子指针为空，便于在树的最左点开始建线索
    if(t) {
        inThreading(t->lchild);//左子树递归线索化
        if(!t-> lchild) { //p的左孩子为空,给p加上左线索,p的左孩子指针指向pre（前驱)
            t->leftTag=1;
            t-> lchild=tempPre;
        } else t->rightTag=0; 

        if(!tempPre->rchild) { //pre的右孩子为空,给pre加上右线索,pre的右孩子指针指向p(后继)
            tempPre->rightTag=1;
            tempPre->rchld=t;
        } else t->rightTag=0;
        tempPre=t;//保持pre指向p的前驱
        inThreading(t->rchild);//右子树递归线索化
    }
}


void inOrderThreading(threadedBinaryTreeNode *head, threadedBinaryTree tt) { //中序遍历二叉树T，并将其中序线索化，Thrt指向头结点
    threadedBinaryTree tempPre = head = new threadedBinaryTreeNode;
    head->rightTag=1;//头结点的右孩子指针为右线索
    head->rchild=tempPre;//初始化时右指针指向自己
    head->leftTag=0;//头结点有左孩子
    if(!tt)head->lchild=tempPre;//若树为空、则左指针也指向自己
    else {
        head->lchild=tt;
        inThreading(tt);//对以tt为根的二叉树进行中序线索化
        tempPre->rightTag=1;
        tempPre->rchild=head;
        head->rchild=tempPre;
    }
}

最优二叉树 哈夫曼树

哈夫曼树的构造

哈夫曼编码

在进行数据压缩时,为了使压缩后的数据文件尽可能短,可采用不定长编码。其基本思想是:为出现次数较多的字符编以较短的编码。为确保对数据文件进行有效的压缩和对压缩文件进行正确的解码，可以利用哈夫曼树来设计二进制编码。

(2)哈夫曼编码:对一棵具有n个叶子的哈夫曼树，若对树中的每个左分支赋予0，右分支赋予1，则从根到每个叶子的路径上，各分支的赋值分别构成一个二进制串，该二进制串就称为哈夫曼编码。

哈夫曼编码是前缀编码。前缀编码:如果在一个编码方案中,任一个编码都不是其他任何编码的前缀,则称编码是前缀编码。前缀编码可以保证对压缩文件进行解码时不产生二义性，确保正确解码。

哈夫曼编码是最优前缀编码。
对于包括n个字符的数据文件，分别以它们的出现次数为权值构造哈夫曼树，则利用该树对应的哈夫曼编码对文件进行编码，能使该文件压缩后对应的二进制文件的长度最短。
这个编码特点符合哈夫曼树的最小权值，保证了编码长度最短。

插入

LL旋转（右单旋转）
由于在结点A的左孩子(L)的左子树(L)上插入了新结点，A的平衡因子由1增至2，导致以A为根的子树失去平衡，需要一次向右的旋转操作。
将A的左孩子B向右上旋转代替A成为根结点,将A结点向右下旋转成为B的右子树的根结点，而B的原右子树则作为A结点的左子树。

RR平衡旋转（左单旋转）
由于在结点A的右孩子(R）的右子树(R）上插入了新结点，4的平衡因子由-1减至-2，导致以A为根的子树失去平衡，需要一次向左的旋转操作。
将A的右孩子B向左上旋转代替A成为根结点，将A结点向左下旋转成为B的左子树的根结点，而B的原左子树则作为A结点的右子树。

LR平衡旋转（左右双旋）
由于在A的左孩子(L)的右子树(R）上插入新结点，A的平衡因子由1增至2，导致以A为根的子树失去平衡，需要进行两次旋转操作，先左旋转后右旋转。
先将A结点的左孩子B的右子树的根结点C向左上旋转提升到B结点的位置，然后再把该C结点向右上旋转提升到A结点的位置。

RL平衡旋转（右左双旋）
由于在A的右孩子(R)的左子树(L)上插入新结点，A的平衡因子由-1减至-2，导致以A为根的子树失去平衡，需要进行两次旋转操作先右旋转后左旋转先将A结点的右孩子B的左子树的根结点C向右上旋转提升到B结点的位置，然后再把该C结点向左上旋转提升到A结点的位置。

特别注意B-tree的阶

m阶B-tree或为空树，或为满足下列条件的m叉树

（1）非叶结点结构
n p0 k1 p1 k2 p2 … km pm

（2）若根结点不是终端结点，则至少有两棵子树，至少一个关键字

（3）除根结点和叶结点外至少有⌈m/2⌉棵子树，至多有m棵子树，至少有⌈m/2⌉-1个关键字，至多有m-1个关键字
（4）B-tree叶结点出现在同一层次上，并且不携带信息，通常称之为失败结点（失败结点并不存在，指向失败结点的指针为空，引入失败结点用于辅助分析B-tree查找性能)

性质

（1）B-tree的高度不包括叶子结点，若n为关键字个数，其范围为（见下面推导）
（2）B-tree插入算法涉及结点分裂，分裂可能导致B-tree高度加一；删除算法涉及结点合并，分裂可能导致B-tree高度减一。
（3）B-tree的阶数为规定值，与B-tree某时刻结点最大关键字数量无关，仅仅是确定一个下限和上限；B-tree的高度不包括叶子结点，B-tree的结点数包括叶子结点。

（4）B-tree中的大部分操作所需的磁盘存取次数与B树的高度成正比

B-tree的高度

（1）B-tree的高度并不包含失败结点，因为失败结点实际并不存在，并不体现于物理位置
（2）B-tree中的大部分操作所需的磁盘存取次数与B树的高度成正比

注意：B-tree的失败结点被认为是叶子结点，但是计算B-tree的高度时并不计入叶子结点

基本操作：创建、查找、插入、删除

创建

B-tree的创建就是不断向树中插入结点的操作

B-tree的查找

B-tree的查找算法包含两个基本操作:1.在B-tree中查找结点，2.在结点内查找关键字
注：由于B-tree常存储在磁盘上，因此第一个查找操作是在磁盘上进行的，而第二个查找操作是在内存中进行的，即先在磁盘中查找到目标结点，在将结点信息读入内存并在结点内采用顺序查找或者折半查找找到关键字，这里的查找算法不唯一，基于关键字有序即可。

1.根据指针查找到某个结点
2.查找结点的关键字序列，查找成功则查找结束，查找失败进入下一步
3.根据关键字信息获得应查找的子树根结点指针，返回第一步，若查找到叶结点时，其对应指针为空指针，则说明树中没有对应的关键字，查找失败
注意：B-tree中所有结点均为实际存储有效信息结点，并不单单作为索引结点

B-tree的插入

基本思想：遵从有序和多路的特点查找定位待插入的位置，插入时遵循平衡的特点进行结点分裂

  1. 定位：基于B-tree查找算法思路，确定关键字插入位置，插入位置一定是最低层中的某个非叶结点
  2. 插入：若插入关键字后待插入结点的关键字个数满足⌈m/2 ⌉-1≦k≦m-1，插入成功，否则进行分裂
  3. 分裂：
    1.将关键字数量超出的结点从⌈n/2 ⌉分裂，左半部分留在原结点，右半部分放入新结点，两结点同层，⌈n/2 ⌉位置关键字插入父结点
    2.若父结点关键字数量超出范围则返回1，表示父节点也需要进行分裂，直至根节点，有可能会导致B-Tree高度加1，否则算法结束

B-tree的删除

基本思想：删除时遵循关键字个数范围[ ⌈m/2 ⌉-1, m-1 ]，若删除后仍满足范围则删除结束，若删除后不满足范围则需要结点借位或结点合并

当被删关键字k不在终端结点（最低层非叶结点）中时，可以用k的前驱（或后继）k’来替代k，然后在相应的结点中删除K，关键字k必定落在某个终端结点中，则转换成了被删关键字在终端结点中的情形

当被删关键字在终端结点（最低层非叶结点）中时：
（1）直接删除关键字，结点关键字数量大于⌈m/2 ⌉- 1，删除关键字该结点后仍满足B-tree对关键字数量的要求
（2）兄弟借位够借，结点关键字数量等于⌈m/2 ⌉- 1，删除关键字后该结点不满足B-tree对关键字的数量要求，与此结点相邻的右（或左）兄弟结点的关键字个数大于⌈m/2 ⌉- 1，则需要使用父子换位法向兄弟结点借位以达到新的平衡
（3）兄弟借位不够借则合并结点，结点关键字数量等于⌈m/2 ⌉- 1，删除关键字后该结点不满足B-tree对关键字的数量要求，且与此结点相邻的左右兄弟结点的关键字个数均等于⌈m/2 ⌉- 1，则将关键字删除后与左（或右）兄弟结点及双亲结点中的关键字进行合并。
在合并过程中，双亲结点中的关键字个数会减1。若其双亲结点是根结点且关键字个数减少至0（根结点关键字个数为1时，有2棵子树），则直接将根结点删除，合并后的新结点成为根；若双亲结点不是根结点，且关键字个数减少到⌈m/2 ⌉- 2，则双亲结点又要与其兄弟结点进行借位或合并操作，重复上述步骤，直至符合B-tree的要求为止

B+Tree

一棵m阶的B+tree，或为空树，或为满足下列特性的m叉树：
（1）树中每个结点至多有m棵子树
（2）若根结点不是叶子结点，则至少有两棵子树
（3）除根之外的所有非终端结点至少有⌈m/2 ⌉棵子树
（4）所有叶结点包含全部关键字及指向相应记录的指针，叶结点中将关键字按大小顺序排列，并且相邻叶结点按大小顺序相互链接起来。
（5）所有非终端结点（可视为索引的索引）中仅包含它的各个子结点（即下一级的索引块）中关键字的最大值及指向其子结点的指针，关键字数量最多为m，子树数量最多为m

m阶的B+tree与m阶的B-tree的主要差异如下:
（1）在B-tree中，具有n个关键字的结点含有n+1棵子树；在B+树中，具有n个关键字的结点只含有n棵子树，即每个关键字对应一棵子树
（2）在B-tree中，根结点关键字个数范围为[ 1, m-1 ]，非终端结点关键字个数范围为[ ⌈m/2 ⌉- 1, m-1 ]；在B+tree中根结点关键字个数范围为[ 2, m]，非终端结点关键字个数范围为[ ⌈m/2 ⌉, m ]
（3）在B-tree中，叶结点为失败结点，仅用于分析B-tree的查找性能，计算B-tree高度时并不计算叶结点层；在B+tree中，叶结点包含关键字和信息，所有叶结点包含了所有关键字和所有信息（即在非叶结点中出现的关键字也会出现在叶结点）
（4）在B-tree中，所有非叶结点包含了所有关键字和所有信息，并且同时作为索引；在B+tree中，所有非叶结点仅起索引作用，非叶结点中的每个索引项只含有对应子树的最大关键字和指向该子树的指针，不含有该关键字对应记录的存储地址，非叶结点中关键字是叶结点中关键字的重复

B+tree的查找

通常在B+树中有两个头指针:一个指向根结点，另一个指向关键字最小的叶结点。可以对B+tree进行两种查找运算:一种是从最小关键字开始的顺序查找，另一种是从根结点开始的多路查找。B+tree多路查找与B-tree不同的是：在查找过程中，非叶结点上的关键字值等于给定值时查找并不终止，而是继续向下查找，直到叶结点上的该关键字为止。故在 B+树中查找时，无论查找成功与否，每次查找都是一条从根结点到叶结点的路径（即使没有要查找的索引也是得查到底才知道没有）。

树、二叉树、森林的转换

树转换为二叉树

树变二叉树的规则：每个结点的左指针指向它的第一个孩子结点。右指针指向它在树中的相邻兄弟结点。
也即：左孩子右兄弟。
根没有兄弟，所以转换以后的树没有右子树。

具体操作：

1.在兄弟之间连线
2.对每一个结点，只保持它与第一个子结点（长子）的连线，与其他子结点的连线全部抹去。
3.以树根为轴心，顺时针旋转45度。

特别注意，对于一般树，其某个结点的子结点的左右次序是相对的，对于二叉树，其某个结点的子结点的左右次序是绝对的。
即若某根结点仅有一个子结点，在一般树中无需区分其左右次序，在二叉树中，必须区分其左右次序。

从树的二叉链表表示的定义可知，任何一棵和树对应的二叉树，其根结点的右子树必空。

二叉树转换为树

二叉树变树的规则：是树变二叉树的逆过程。

1.加线。如果某个结点有左孩子，那么把这个左孩子的右孩子，右孩子的右孩子，一直右下去，全部连接到这个结点。这个对应树变二叉树算法中的抹掉与长子以外的结点的连线，现在要找回自己的全部儿子。
2.去线。删除树中所有结点与这些右孩子的连线。找回了自己的儿子，不容许别人还和它们有瓜葛。
3.层次调整。

二叉树转换为森林

二叉树转换为森林的规则：

1.递归出口：若二叉树非空，则二叉树根及其左子树作为第一棵树的二叉树形式。
2.递归：操作二叉树的右子树。即拿掉一棵树后，再对剩下的二叉树部分进行同样的操作，直到无右子树为止。
3.砍成一堆二叉树后，还得注意并没结束，重点不是二叉树，而是树，因此用上面的二叉树化树的算法化成森林。

森林转换为二叉树：

森林转换为二叉树的规则：

先将每一棵树化为二叉树
第一棵树的根作为转换后的二叉树的根
第一棵树的左子树作为转换后的二叉树的根的左子树
第二棵树作为二叉树的右子树
第三棵树作为二叉树右子树的右子树
…
这里主要运用的点是：转换为二叉树后，这个二叉树没有右子树，因此腾出右手可以接一棵新的树，因此这样可以连接很多由树化来的二叉树。

对照这个算法去想二叉树变森林，就很容易明白，先砍掉根和它的左子树作为一个整体，再对剩下的部分同样操作。
