排序

予早 2024-12-01 21:58:13
Categories： Tags：

基本概念

排序稳定性：关键字相同的元素间在排序前和排序后相对位置不变则称排序算法稳定

在排序过程中，根据数据元素是否完全在内存中，可将排序算法分为两类：

  1. 内部排序，是指在排序期间元素全部存放在内存中的排序，内部排序的主要操作是比较和移动。
  2. 外部排序，是指在排序期间元素无法全部同时存放在内存中，必须在排序的过程中根据要求不断地在内、外存之间移动的排序。

内部排序

根据逐步扩大记录有序序列长度的原则不同，可以将内部排序分为以下几类。

(1)插人类:将无序子序列中的一个或几个记录“插入”到有序序列中，从而增加记录的有序子序列的长度。

(2)交换类:通过“交换”无序序列中的记录从而得到其中关键字最小或最大的记录，并将它加入到有序子序列中，以此方法增加记录的有序子序列的长度。

(3)选择类:从记录的无序子序列中“选择”关键字最小或最大的记录，并将它加入到有序子序列中，以此方法增加记录的有序子序列的长度。

(4)归并类:通过“归并”两个或两个以上的记录有序子序列，逐步增加记录有序序列的长度。

(5)分配类:是唯一一类不需要进行关键字之间比较的排序方法，排序时主要利用分配和收集两种基本操作来完成。

外部排序

基本思想：①子文件内部排序形成归并段，②归并段逐趟排序形成有序文件

注意：默认排序目标为非递减有序序列

算法应该从以下四个方面描述：算法思想（基本思想）、算法步骤、算法实现、算法分析

内部排序

插入

直接插入排序

算法思想

将待排序序列中的元素依次通过顺序查找确定位置并插入已排序完成的子序列直至整个排序完成。

算法步骤
  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 排序完成的子序列为[nums[0], …, nums[i-1]]，待排序序列为[nums[i], …, nums[n-1]]。
  3. 选择待排序序列第一个元素item = nums[i]，顺序查找该元素在排序完成的子序列中的插入位置k。
  4. 将nums中[nums[k], …, nums[i-1]]序列整体向后移动一位，占据原来nums[i]的位置，然后把item 值放到nums[k]的位置。
  5. 此时，排序完成的子序列变为[nums[0], …, nums[i]]，待排序序列变为[nums[i+1], …, nums[n-1]]，由此重复执行3~4步骤直至带排序序列为空。
算法实现
nums = [49, 38, 65, 97, 76, 13, 27, 49]


def direct_insertion_sort(nums):
    n = len(nums)
    for i in range(1, n):
        item = nums[i]
        j = i - 1
        # 比较和移动同时进行以精简代码
        while j >= 0 and nums[j] > item:
            nums[j + 1] = nums[j]
            j -= 1
        nums[j + 1] = item


direct_insertion_sort(nums)
print(nums)  # [13, 27, 38, 49, 49, 65, 76, 97]
算法分析

排序稳定性：稳定。取决于顺序查找的实现，由于要插入的元素是从左往右依次的，有相等元素时，若a的等值元素已经在已排序序列中，则顺序查找必须查找到a的所有等值元素右边的第一个位置才能保证排序稳定性，经典的方式是倒序找到第一个不大于要插入元素的位置，若查找到的是所有等值元素中间或者最左边的第一个位置，则排序不稳定。

时间复杂度：$O(n^2)$。数组长度为n，则共会进行n-1次将元素放到正确位置的操作，每次操作都会查找正确的位置，使用顺序查找则查找次数随已排序子序列长度递增，耗时$O(n)$，然后将已排序序列中正确位置以及后面的元素整体后移动一位，耗时为$O(n)$，则整体耗时 $O((n-1) * (n + n)) = O(n^2)$。

空间复杂度：$O(1)$。仅使用额外常数空间，空间复杂度为$O(1)$。

折半插入排序

算法思想

将待排序序列中的元素依次通过折半查找确定位置并插入已排序完成的子序列直至整个排序完成。就是在直接插入排序的基础上以折半查找替换顺序查找。

算法步骤
  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 排序完成的子序列为[nums[0], …, nums[i-1]]，待排序序列为[nums[i], …, nums[n-1]]。
  3. 选择待排序序列第一个元素item = nums[i]，折半查找该元素在排序完成的子序列中的插入位置k。
  4. 将nums中[nums[k], …, nums[i-1]]序列整体向后移动一位，占据原来nums[i]的位置，然后把item 值放到nums[k]的位置。
  5. 此时，排序完成的子序列变为[nums[0], …, nums[i]]，待排序序列变为[nums[i+1], …, nums[n-1]]，由此重复执行3~4步骤直至带排序序列为空。

算法分析

排序稳定性：稳定。取决于折半查找的实现，由于要插入的元素是从左往右依次的，有相等元素时，若a的等值元素已经在已排序序列中，则折半查找必须查找到a的所有等值元素右边的第一个位置才能保证排序稳定性，若查找到的是所有等值元素中间或者最左边的第一个位置，则排序不稳定。

时间复杂度：$O(n^2)$。数组长度为n，则共会进行n-1次将元素放到正确位置的操作，每次操作都会查找正确的位置，使用折半查找则查找次数随已排序子序列长度递增，耗时$O(\log{n})$，然后将已排序序列中正确位置以及后面的元素整体后移动一位，耗时为$O(n)$，则整体耗时 $O((n-1) * (\log{n} + n)) = O(n^2)$。

空间复杂度：$O(1)$。仅使用额外常数空间，空间复杂度为$O(1)$。

希尔排序（shell，缩小增量排序）

算法思想

设置一个递减至1的步长序列$[d_1,d_2,…,d_n,1]$，将待排序序列根据步长分为若干子序列[i,i+d,i+2d,…,i+kd]并对每个每个子序列分别进行直接插入排序，依次对步长表中每一个步长进行上述操作，直至步长为1的操作完成时排序结束。

算法1步骤

算法1步骤是排序的标准思路

  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 取步长序列$2^m-1，m为正整数且2^m-1<=n/2$，从大到小排序
  3. 取步长序列中下一个步长d
  4. 对每一个子序列各进行一次直接插入排序
  5. 返回步骤3重复直到步长序列为1的操作完成
算法1实现
import math

nums = [49, 38, 65, 97, 76, 13, 27, 49]


def shell_sort(nums):
    n = len(nums)
    m = int(math.log((n + 1) // 2 + 1, 2))
    # 每个步长处理一次
    for _m in range(m, 0, -1):
        d = 2 ** _m - 1
        # 每个步长中的每个子序列处理一次
        for i in range(0, d):
            # 每个子序列进行一次直接插入排序，子序列元素从i开始，步长为d，i==0，d==1时就是标标准准的直接插入排序
            for j in range(i + d, n, d):
                item = nums[j]
                k = j - d
                while k >= i and nums[k] > item:
                    nums[k + d] = nums[k]
                    k -= d
                nums[k + d] = item


shell_sort(nums)

print(nums)
算法2步骤

算法2等价于算法1，是一种能让代码更加精简的思路

  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 取步长序列$2^m-1，m为正整数且2^m-1<=n/2$，从大到小排序
  3. 取步长序列中下一个步长d
  4. 对每一个子序列各进行一次直接插入排序中的一趟
  5. 返回步骤4，直到每个子序列的最后一趟直接插入排序完成
  6. 返回步骤3重复直到步长序列为1的操作完成
算法2实现
def shell_sort(nums):
    n = len(nums)
    m = int(math.log((n + 1) // 2 + 1, 2))
    # 每个步长处理一次
    for _m in range(m, 0, -1):
        d = 2 ** _m - 1
        # 下面是上面的等价算法，下面相当于每个子序列执行一次折半插入中的一趟，然后再执行一趟
        for i in range(d, n):
            j = i
            while j >= d and nums[j] < nums[j - d]:
                nums[j], nums[j - d] = nums[j - d], nums[j]
                j -= d
算法分析

排序稳定性：不稳定。两个等值元素被分到两个子序列，这样有可能两个子序列分别进行直接插入排序后导致位置反转。

时间复杂度：希尔排序的时间复杂度与其步长序列的选取有关。

步长序列 最坏情况下时间复杂度
$n/2^i$ $O(n^2)$
$2^m-1$ $O(n^{3/2}$
$2^i3^j$ $O(n\log^2n)$

空间复杂度：$O(1)$。仅使用额外常数空间，空间复杂度为$O(1)$。

选择

直接选择排序

算法思想

通过顺序查找从待排序序列中选择最小的元素与已排序完成的子序列之后的一个元素置换。

算法步骤

  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 排序完成的子序列为[nums[0], …, nums[i-1]]，待排序序列为[nums[i], …, nums[n-1]]。
  3. 从待排序序列中获取最小元素nums[k]，与已排序序列的后一个位置也就是待排序序列第一个位置元素nums[i]进行交换
  4. 此时，排序完成的子序列变为[nums[0], …, nums[i]]，待排序序列变为[nums[i+1], …, nums[n-1]]，由此重复执行3步骤直至待排序序列为空，经过n-1趟即可排序完成。

算法实现

nums = [49, 38, 65, 97, 76, 13, 27, 49]


def direct_selection_sort(nums):
    n = len(nums)
    for i in range(n - 1):
        idx = i
        for j in range(i, n):
            if nums[idx] > nums[j]:
                idx = j
        nums[idx], nums[i] = nums[i], nums[idx]


direct_selection_sort(nums)

print(nums)

算法分析

排序稳定性：不稳定。由于会将待排序序列中的最小元素与第一个元素位置进行交换，若第一个元素a和要交换到的位置i之间有一个a的等值元素，这样就导致等值元素相对位置发生变化。当然，若在经典直接选择排序上进行修改，利用冒泡思想，让最小元素不断前移直到待排序序列的第一个位置，这样每趟交换一次就变成多次，时间复杂度不变，但常系数会变为原来两倍。

时间复杂度：$O()$。数组长度为n，会进行n-1趟排序操作，每趟操作要遍历整个待排序序列以及交换一次值，交换值的操作在渐进意义下可以忽略，则时间复杂度为$O(n+(n-1)+…+2)=O(\frac{n+2}{2}(n-1))=O(n^2)$。

空间复杂度：$O(1)$。仅使用额外常数空间，空间复杂度为$O(1)$。

树形选择排序（锦标赛排序）

锦标赛排序的名字来源于单败淘汰制的竞赛形式，该赛制中选手两两比较，败者立即淘汰，胜者进入下一轮，这种淘汰方式冠军一定是最优秀的，但最后一轮比赛中被淘汰的选手不一定是第二好的，有可能具有亚军实力的人在第一轮就被冠军淘汰了。

算法思想

将待排序序列构造为完全二叉树，进行n次锦标赛，决出n次冠军，排序完成。

算法步骤

最小树形排序为例。

待排序元素是叶子节点显示的元素。红色边显示的是每一轮比较中较小的元素的胜出路径。显然，完成一次＂锦标赛＂可以选出一组元素中最小的那一个。

每一轮对$n$个元素进行比较后可以得到$\frac{n}{2}$个「优胜者」，每一对中较小的元素进入下一轮比较。如果无法凑齐一对元素，那么这个元素直接进入下一轮的比较。

完成一次「锦标赛」后需要将被选出的元素去除。直接将其设置为无穷大，然后再次举行「锦标赛」选出次小的元素。

之后一直重复这个操作，直至所有元素有序。

算法实现

算法分析

时间复杂度：$O(n\log{n})$。筹备锦标赛需要$O(n)$时间，每一次锦标赛需要比较$\log{n}$次，一共需要比赛n次，则时间复杂度为$O(n+n\log{n})=O(n\log{n})$

空间复杂度：$O(n)$。需要额外使用一个最低层结点数量为n的完全二叉树，空间复杂度为$O(n)$。

排序稳定性：不稳定

交换

冒泡（交换）排序

算法思想

从前往后（或从后往前）两两比较相邻元素的值，若为逆序则交换，直到序列比较完毕完成一趟冒泡，最大的元素被放到最后的位置（或最小的元素被放到最开始的位置），一趟冒泡可将一个元素放到最终位置，最多n-1次冒泡排序完成

算法步骤

  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 待排序序列为[nums[0], …, nums[i-1]]，排序完成的子序列为[nums[i], …, nums[n-1]]。
  3. 对待排序序列从前往后两两比较相邻元素的值，若为逆序（即nums[i]<nums[i+1]）则交换两者的值
  4. 回到步骤3，直到比较待排序序列长度减一次时当前最大的元素将被放置到待排序序列最后一个位置，一趟冒泡完成
  5. 此时，待排序序列变为nums[0], …, nums[i-2]]，排序完成的子序列变为[[nums[i-1], …, nums[n-1]]，由此重复执行3~4步骤直至待排序序列为空，经过n-1趟即可排序完成。

算法实现

nums = [49, 38, 65, 97, 76, 13, 27, 49]


def bubble_swap_sort(nums):
    n = len(nums)
    # 循环 n - 1 次
    for i in range(n - 1, 0, -1):
        # 每次从前到后通过交换的方式将最大的元素放到最后面
        for j in range(i):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]


bubble_swap_sort(nums)
print(nums)

算法分析

排序稳定性：稳定。若相邻元素两两对比时相等时不发生交换，则等值元素相对位置不会变，则排序稳定，若相邻元素两两对比相等时发生交换，则等值元素相对位置会发生变化，排序不稳定。

时间复杂度分析：$O(n^2)$。冒泡排序会进行n-1趟，每一趟要操作的待排序序列长度分别为[n,n-1,…,2]，每一趟会进行待排序序列减一的比较和不超过比较次数的交换，故时间复杂度为$O((n-1)+(n-1-1)+…+1)=O(\frac{(n-1)+1}{2}*(n-1))=O(n^2)$。

空间复杂度：空间复杂度：$O(1)$。仅使用额外常数空间，空间复杂度为$O(1)$。

快速（交换）排序

快速(交换)排序
算法思想

从待排序序列中选择一个元素作为中轴（该元素称为基准数），将所有小于中轴的数交换到中轴右边，将大于中轴的数交换到中轴右边，这样基准数所在位置就是最终排序结果的位置，而对于小于中轴和大于中轴的两个待排序子序列，重复上述步骤确定每一个基准数位置，无限细分直至子序列长度为1则排序完成。

注意：

  1. 通常基准数会选择数组第一个元素，基准数不是第一位的情况，基准数和第一位交换就行了
  2. 上述算法思想中没有体现元素相等时应当放到左子序列还是右子序列，两者均可，不会影响排序结果
  3. 这里的交换通常基于双指针实现

算法步骤

  1. nums为长度为n的数字序列，序列索引为[0, n-1]。
  2. 待排序序列为nums。
  3. 从待排序序列中选择第一个元素作为中轴基准数，赋给中间变量x，此时，待排序序列第一个位置可以用于存储小于中轴基准数的元素。
  4. 基于双指针开始找值。
  5. 找小于等于元素往左放。从右到左遍历找到第一个小于中轴基准数的数，将该数放置到之前空出来用于存放小于中轴基准数的位置，这时原位置空出来可以用于存储大于等于中轴基准数的元素。
  6. 找大于等于元素往右放。从左到右找到第一个大于等于中轴基准数的数，将该数放置到之前空出来用于存放大于等于中轴基准数的位置，这时原位置空出来可以用于存储小于中轴基准数的元素。
  7. 回到步骤5~6开始依次一小一大交换，直到双指针指向同一个位置，即中轴基准数的排序最终位置，该位置值替换为中间变量x中的值，此时一趟快速排序完成。
  8. 中轴基准数将此时的待排序序列分为小于中轴的子序列和大于等于中轴的子序列，基于递归思想，左右子序列分别重复3~7步骤无限细分直至子序列长度为1则排序完成。递归出口就是左子序列和右子序列都是只含有一个元素的时候。

注意：上述步骤中将与中轴基准数相等的元素放到了右子序列中。

算法实现

要强调的是，在下面实现中，采用的是记录然后覆盖基准数，最后确定基准数的位置后再赋值的方式。所以对于本实现方法，如果基准数在左边第一位，应该是先从右向左遍历找一个小于基准数的数替换基准数。当然，如果基准数在右边第一位，应该先从左边遍历找一个大于基准数的数，也可以将基准数与第一位置的数互换，这样还是先从左进行遍历。不过这样很麻烦，不如默认第一位。

nums = [49, 38, 65, 97, 76, 13, 27, 49]


def quick_swap_sort(nums, left, right):
    # 1.基准数取最左边的
    x, i, j = nums[left], left, right
    # 2.确定基准数的位置
    while i < j:
        # 2.1从右向左找第一个小于x的数
        while i < j and nums[j] >= x:
            j -= 1
        # 2.2替换序列右边的一个不小于基准值的值
        if i < j:
            nums[i] = nums[j]
            i += 1
        # 2.3从左向右找第一个大于等于x的数
        while i < j and nums[i] < x:
            i += 1
        # 2.4替换序列左边的一个大于基准值的值
        if i < j:
            nums[j] = nums[i]
            j -= 1
        # 2.5基准数确定位置后的赋值
        nums[i] = x
        # 2.6左子序列不只有一个元素时递归
        if left < i - 1:
            quick_swap_sort(nums, left, i - 1)
        # 2.7右子序列不只有一个元素时递归
        if right > i + 1:
            quick_swap_sort(nums, i + 1, right)


quick_swap_sort(nums, 0, len(nums) - 1)

print(nums)

void quickSort(int a[],int l,int r) {
    int x=a[l],i=l,j=r;//基准数取最左边的
    while(i<j) {//确定基准数的位置
        while(i < j && a[j] >= x)j--; //从右向左找第一个小于x的数
        if(i < j)a[i++] = a[j];//替换序列右边的一个不小于基准值的值
        while(i < j && a[i] < x)i++; //从左向右找第一个大于等于x的数
        if(i < j)a[j--] = a[i];//替换序列左边的一个大于基准值的值
    }
    a[i] = x;//基准数确定位置后的赋值
    if(l<i-1)quickSort(a,l,i-1);//左子序列不只有一个元素时递归
    if(i+1<r)quickSort(a,i+1,r);//右子序列不只有一个元素时递归
}

算法分析

排序稳定性：不稳定。快速（交换）排序的中轴基准数是任意的，由此可能会导致等值元素被选中为中轴基准数，假设三个等值元素已中间的元素作为中轴基准数，那么必然导致等值元素相对位置变化，另外中轴基准数两边的交换也是随机的，假设两个小于中轴基准数的元素均在中轴基准数右侧，则会先将第二个等值元素交换到中轴基准数左侧，等到第一个等值元素交换到左侧时，两者相对位置就发生了变化。

时间复杂度：$O(n\log{n})$。可带入主定理直接算得或者推导一下。第一趟选取一个中轴基准数确定一个元素的位置，得到两个子序列，第二趟，两个子序列选取两个中轴基准数确定两个元素的位置，得到四个子序列，依此类推。

空间复杂度：$O(\log{n})$。由于快速（交换）排序使用了递归，递归树最大深度为$\log{n}$，每次调用除递归部分仅占用常数空间，故空间复杂度为$O(\log{n})$。

二路归并排序

算法思想

将待排序序列视为n个有序子序列，两两归并为一个有序子序列得到⌈n/2 ⌉个有序子序列，重复这个过程直至合并为一个有序序列，排序完成

算法步骤
算法实现
def merge(nums, left1, right1, left2, right2):
    i, j = left1, left2
    k = 0
    # 每次调用完本函数之后内存回收，内存中最多只会有一个merge_space占用内存
    merge_space = [0] * (right2 - left1 + 1)
    while i <= right1 and j <= right2:
        if nums[i] <= nums[j]:
            merge_space[k] = nums[i]
            i += 1
        else:
            merge_space[k] = nums[j]
            j += 1
        k += 1

    while i <= right1:
        merge_space[k] = nums[i]
        i += 1
        k += 1
    while j <= right2:
        merge_space[k] = nums[j]
        j += 1
        k += 1

    start = 0
    for i in range(left1, right2 + 1):
        nums[i] = merge_space[start]
        start += 1

# 迭代倍增方式
def two_way_merge_sort(nums):
    seg, n = 1, len(nums)
    while seg < n:
        for i in range(0, n, 2 * seg):
            left1, right1 = i, i + seg - 1
            left2, right2 = right1 + 1, right1 + seg
            merge(nums, left1, right1, left2, right2)
        seg <<= 1


two_way_merge_sort(nums)
print(nums)


nums = [49, 38, 65, 97, 76, 13, 27, 49]

# 递归方式
def two_way_merge_sort(nums, left, right):
    if right - left < 1:
        return
    # 分解
    mid = (right + left) // 2
    two_way_merge_sort(nums, left, mid)
    two_way_merge_sort(nums, mid + 1, right)
    # 合并
    merge(nums, left, mid, mid + 1, right)


two_way_merge_sort(nums, 0, len(nums) - 1)
print(nums)
算法分析

时间复杂度：$O(n\log{n})$。无论是迭代还是递归，终归要执行$O(\log{n})$趟排序，而每趟排序中任意两个子序列合并耗时都与两个子序列长度之和成正比，merge本身时间复杂度就是$O(n)$，则任意两个子序列合并耗时可以表示为$O(\frac{n}{k})$，其中$k=\frac{序列总长度}{两个子序列总长度}$，所以任意一趟合并，耗时都与整个序列长度成正比，为$O(n)$，则时间复杂度为$O(n\log{n})$。

空间复杂度：$O(n)$。递归部分占用空间复杂度为$O(\log{n})$，在整个执行过程中，只会同时执行一个merge方法，该方法使用的额外空间merge_space只会存在一份，占用$O(n)$，空间复杂度为$O(\log{n}+n)=O(n)$。

排序稳定性：稳定。经典二路归并排序是稳定的，由于改变元素顺序是发生在merge中，其采用额外空间以及相适应的大小比较进行排序合并的方式能保证稳定性。

分配

计数排序

算法思想

基于待排序序列构建一个记录排序后元素位置的辅助数组，构建一个与待排序序列等长的排序结果数组，倒序遍历待排序序列并通过辅助数组找到对应位置放到结果数组中直至遍历完毕完成排序。

注意：

  1. 辅助数组是通过计算每个元素出现次数，再计算前缀和构建成的。
  2. 逆序遍历待排序序列、构建的结果数组的原因一是为保证每个元素均可呈现在结果中，另外就是为保证排序稳定性。因为实际场景中排序的并非是单纯的数字，而是一个个对象，会根据对象的某个或者某几个属性进行排序，辅助数组只能体现出排序属性的大小，没办法通过辅助数组获取到原本的元素，除非就是单纯的数字才有这种对应关系，所以计数排序中需要一个专门结果数组保存排序结果。
  3. 若待排序序列中是纯数字，则可以考虑节省结果数组的空间消耗。
  4. 计数排序天然适合非负整数排序，若要排序的有负整数，则可以考虑设置一个偏移量转换为非负整数，若有小数，则可以考虑乘以$10^n$，若为对象时，需考虑一般情况，假设对象序列为$[a_1,a_2,a_3,…a_n]$，根据a的属性key排序，则有$[key_1,key_2,key_3,…,key_n]$，必须将key序列每个元素都唯一映射到0-k才能构建辅助数组，若无法一一映射则不能使用计数排序。
  5. 计数排序是非原地排序算法

算法步骤

  1. 求出数组中的最大最小值
  2. 计算每个数出现次数
  3. 求出每个数出现次数的前缀和
  4. 利用出现次数的前缀和定位元素位置，逆序遍历原数组并将对象元素放入到结果数组中，返回结果数组

算法实现

nums = [49, 38, 65, 97, 76, 13, 27, 49]


def counting_sort(nums):
    # 找出数组中的最大值和最小值
    max_val = max(nums)
    min_val = min(nums)
    k = max_val - min_val + 1

    # 初始化计数数组
    count = [0] * k

    # 计算每个元素的出现次数
    for num in nums:
        count[num - min_val] += 1

    # 在计数数组基础上构建前缀和数组，此时前缀和就是每个原数字的索引
    for i in range(1, k):
        count[i] += count[i - 1]

    # 构建排序后的数组
    output = [0] * len(nums)
    for num in reversed(nums):
        idx = count[num - min_val] - 1  # 通过一个辅助数组计数、求前缀和从而得到索引
        output[idx] = num
        count[num - min_val] -= 1

    return output


res = counting_sort(nums)
print(res)

算法分析

排序稳定性：稳定。前缀和、逆序遍历和结果数组三者保证了排序的稳定性。

时间复杂度：$O(n+k)$。找出数组中最大值和最小值耗时$O(n)$，统计频次耗时$O(n)$，计算前缀和耗时$O(k)$，构建结果数组耗时$O(n)$，时间复杂度为$O(n+n+k+n)=O(n+k)$。

空间复杂度：$O(n+k)$。计数排序使用长度为k的空间作为辅助数组，使用长度为n的空间存储结果，空间复杂度为$O(n+k)$

考虑一种特殊情况。给定数组nums，其中元素取值范围为0~65535，设计一个时间复杂度为O(n)，空间复杂度为O(1)的算法专门用于这类数组排序。

基于计数排序，进行简化，辅助数组使用定长65536，不使用结果数组。

nums = [49, 38, 65, 97, 76, 13, 27, 49]


def counting_sort(nums):
    # 初始化计数数组
    k = 65536
    count = [0] * k

    # 计算每个元素的出现次数
    for num in nums:
        count[num] += 1

    # 顺序遍历count修改nums的值
    i, j = 0, 0
    while i < len(count):
        while count[i] > 0:
            nums[j] = i
            count[i] -= 1
            j += 1
        i += 1

基数（分配）排序

算法思想

单个关键字转换为多关键字然后进行排序，线性表nums中每个元素任意一个元素$a_i$的关键字由d维度元组$(k_i^0,k_i^1,…,k_i^{d-1})$组成，且$k_i^j$满足$0<=k_i^j<r-1(0<=i<n,0<=j<d-1)$，元素从左到右权重依次减小，根据d个关键字进行排序，r叫做基数。

基数排序对d维度元组中每个维度$k^j$进行一趟排序，每一趟排序根据$k_i^j$的取值进行”分配“，会被分配为最多r组，将r组依次”收集“，完成一趟排序，随后根据下一个维度进行排序，直至所有维度均参与过一次排序。

联系计数排序的一个理解方式：由于元素大小相差较大不能直接使用计数排序（例如元素取值范围为[0,10^8)），于是将每个元素分为d个维度，每个维度的取值范围相同为[0,r)，这个范围远远小于元素本身的取值范围，此时只需要进行d次计数排序即可，r就是计数排序中辅助数组的长度，也为基数排序的基数，。

基于不同排序顺序，计数排序分为：

  • LSD，Least significant digital：最低有效位，按关键字权重递减排序。

  • NSD，Most significant digital，最高有效位，按关键字权重递增排序。

def counting_sort(nums, exp):
    n = len(nums)

    count = [0] * 10

    for i in range(n):
        index = nums[i] // exp
        count[index % 10] += 1

    for i in range(1, 10):
        count[i] += count[i - 1]

    output = [0] * n
    i = n - 1
    while i >= 0:
        index = nums[i] // exp
        output[count[index % 10] - 1] = nums[i]
        count[index % 10] -= 1
        i -= 1

    for i in range(n):
        nums[i] = output[i]


def radix_sort(arr):
    max_val = max(arr)
    exp = 1
    while max_val // exp > 0:  # 一共会进行d次
        counting_sort(arr, exp)
        exp *= 10


arr = [170, 45, 75, 90, 802, 24, 2, 66]
radix_sort(arr)
print(arr)

外部排序

算法思想

外部排序是相对于内部排序而言的，内部排序是指整个排序过程在内存内部，而外部排序则是由于需要排序的数据过多，无法将所有数据一次性放入内存进而排序，需要先将数据放在外存，排序时再把数据一部分一部分调入内存进行排序，在排序过程中需要多次进行外存和内存之间交换，外部排序常采用归并排序方式

算法步骤

  1. 归并段初始生成：根据内存缓冲区大小，采用一定方法将所有数据分割并分别进行内部排序获得若干有序归并段
  2. 归并段外部归并：对有序归并段逐趟归并进而扩大有序归并段直至归并为一整个有序段

注意：归并段归并时需要将数据进行外存与内存的交换

算法实现

  1. 均等分割+外部多路归并
  2. 均等分割+败者树优化的外部多路归并
  3. 置换选择排序分割+最佳归并树优化的归并段多路归并

注：最佳归并树，即是m叉哈夫曼树

算法分析

时间复杂度：分割预处理内部排序所需时间+外部归并所需时间+外存读写时间。由于外存读写时间远大于另外两者，所以时间复杂度集中在外存读写时间上，可通过减少I/O次数增加时间效率。

空间复杂度：外部多路归并路数*缓冲区大小。

排序稳定性：稳定。

均等分割与内部排序+外部多路归并

算法思想

  1. 均等分割，将外存上的文件均等分割为若干子文件；内部排序，使用合适内部排序算法对每个子文件排序获得若干子文件（归并段）。
  2. 外部归并，将n个归并段读入内存，进行n路归并，在一趟归并中，输出缓冲区满后，写回内存，输入缓冲区中数据排序完成重新读入下一个归并段直至本趟归并完成。

注：输入输出缓冲区大小一般与归并段大小相等

算法分析

时间复杂度：分割预处理内部排序所需时间+外部归并所需时间+外存读写时间。由于外存读写时间远大于另外两者，所以时间复杂度集中在外存读写时间上，可通过减少I/O次数增加时间效率。I/O次数取决于归并排序趟数，每一趟归并排序都会对所有数据读写各一次，若r为初始归并段数量，k为归并路数，则归并趟数为$log_kr$。可以通过增加归并路数k或者减少初始归并段段数r来减少外存I/O已到达提高外部排序速度的目的。简而言之，增加路数，减少段数。增加路数意味着输入缓冲区数量要增加，减少段数意味着单个输入缓冲区大小要增加，本质都是在增加内存占用，是一种空间换时间的思路。

空间复杂度：外部多路归并路数*缓冲区大小。

排序稳定性：稳定。

均等分割与内部排序+败者树优化的外部多路归并

算法思想

增加归并路数，可减少归并趟数，进而减少磁盘IO次数，从而降低外部排序总时间消耗。

由于增加归并路数，内部归并排序时间将增加，k路归并排序，在k个元素中选择关键字最小的元素需要比较k-1次，每趟归并n各元素需要$(n-1)(k-1)$次比较，则整个外部多路归并过程中，需要进行$log_kr$趟，内部归并排序总比较次数为$log_kr(n-1)(k-1)=\frac{log_2r(n-1)(k-1)}{log_2k}$，其中$\frac{k-1}{log_2k}$随着k增长而增长，因此内部归并时间也随k增长而增长，也就是说路数k的增长一方面减少了IO次数提高了排序速度，另一方面增加了内部排序的耗时，导致减少IO的效益有所折扣。

使用败者树优化归并排序。败者树对k各元素进行比较，比较次数就是其高度$\log_2k$，则基于败者树外部多路归并总比价次数为$log_kr(n-1)\log_2k=log_2r(n-1)$，此时比较次数与k无关，因此只要内存空间允许，增加归并路数k将有效减少IO次数，提高排序速度。

归并路数k并不能无限增大，归并路数增大，需要相应增加输入缓冲区个数，若可使用内存空间不变，则此时必然需要减少输入缓冲区容量，此时归并段r反而会增多（缓冲区与归并段大小保持一致），进而影响$log_rk$的值而影响IO次数，当内存空间有限时，需要保证r和k之间的合理搭配。
